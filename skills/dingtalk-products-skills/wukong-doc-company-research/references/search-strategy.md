# 企业调研信息收集策略

> 各维度的搜索查询方案和 draft.md 写入模板。

## A1. 企业基础画像

### web_search 查询方案

**阶段一：企业档案**
```
web_search: "{公司} 企业注册信息 创始团队 发展沿革"
web_search: "{公司} 公司背景 工商信息 成立时间"
```

**阶段二：资本动态**
```
web_search: "{公司} 融资轮次 投资方 估值变化"
web_search: "{公司} 上市时间 股票代码 证券简称"
```

**阶段三：核心管理层**
```
web_search: "{公司} 法定代表人 实控人 董事长"
web_search: "{公司} 高管名单 管理层履历 股权结构"
```

### web_fetch 深度抓取

```
web_fetch(url: "{公司官网}/about-us") — 获取企业简介、成立年份、总部位置、发展里程碑
web_fetch(url: "{招股说明书/年度报告链接}") — 获取股权分布、历史融资、重要股东信息
```

## A4. 经营绩效指标

### web_search 查询方案

**阶段一：财务表现**
```
web_search: "{公司} 2024年度财报 收入 净利润 增长率"
web_search: "{公司} 年度报告 利润率 毛利率 净利率"
```

**阶段二：运营指标**
```
web_search: "{公司} 财务指标 ROE 资产收益率"
web_search: "{公司} 研发费用 研发投入比 人均产出"
```

**阶段三：趋势研判**
```
web_search: "{公司} 财务趋势 同比变化 环比变化"
web_search: "{公司} 业绩展望 分析师评级 投资建议"
```

### web_fetch 深度抓取

```
web_fetch(url: "{官方财报页面}") — 获取近三年完整财务数据
web_fetch(url: "{券商研报链接}") — 获取盈利预测、目标价、评级
```

### 数据校验机制

- 财务数据需至少 2 个独立来源交叉验证（如官方年报 + 第三方数据平台）
- 数据差异超过 10% 需说明原因
- 优先采用公司官方披露的财报数据

## A6. 内部资料补充（可选）

通过钉钉文档检索已有的内部研究资料：

```
dws doc search --query "{目标公司} 研究" --format json
dws doc search --query "{目标公司} 行业分析" --format json
```

IF 搜索到相关文档 → 调用 `dws doc read --node <NODE_ID> --format json` 获取内容
ELSE → 跳过此步骤，标注"未找到内部研究资料"
