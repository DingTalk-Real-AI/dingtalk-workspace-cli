# 质量验收标准与问题排查指南

## 信息采集阶段验收标准

- [ ] 企业基础档案：注册成立日期、总部所在地、核心创始团队、人员规模等关键信息已完整收录
- [ ] 资本运作轨迹：完整融资轮次记录、上市状态及时间节点、当前估值或市场估值已核实
- [ ] 发展里程碑：至少梳理出 5 个具有代表性的关键发展节点
- [ ] 业务组合全景：识别全部主营业务板块，获取各板块营收贡献比例
- [ ] 产品体系清单：核心产品目录完整，收集用户规模或市场占有率数据
- [ ] 营收构成分析：按业务线、区域、客户类型的收入分布情况已获取（至少包含业务维度）
- [ ] 市场定位评估：获取行业排名位置及市场份额数据
- [ ] 竞争对手画像：完成至少 3 家主要竞品的多维度对比分析表
- [ ] 核心竞争力：识别并评估竞争壁垒的类型及强度
- [ ] 财务核心指标：近 3 个财年的营业收入、净利润、毛利率等关键数据已采集
- [ ] 运营效能指标：ROE、ROA 等效率类指标已获取（如公开可得）
- [ ] 数据校验机制：财务数据至少通过 2 个独立来源交叉核实
- [ ] 数值一致性校验：百分比类数据（如市场份额）加总不超过 100%，同口径数据无矛盾
- [ ] 非上市/无融资说明：若公司无独立融资或上市记录，已在对应章节显式说明原因而非直接跳过
- [ ] 近期动态追踪：近 6 个月内的重要事件或新闻已收集
- [ ] 辅助资料检索：已完成钉钉文档库搜索（无论结果如何均需记录）
- [ ] 所有数据项均已标注信息来源及时间戳
- [ ] draft.md 文档内容完整，涵盖六大分析维度

## 分析深度验收标准

- [ ] SWOT 框架分析已完成，每个象限至少包含 2 条有事实依据的关键点
- [ ] 竞争护城河评估已完成，给出明确的护城河等级评定
- [ ] 风险矩阵分析已完成，对关键风险点进行深度剖析
- [ ] 综合评估结论及评分已完成
- [ ] 所有分析性结论均具备数据或事实支撑

## 文档输出验收标准

- [ ] 调研摘要要素完整：包含一句话核心结论及 3-5 条关键发现
- [ ] 章节名严格遵循 report-template.md 模板定义（核心洞察/企业画像/商业版图/竞争力分析/财务健康度/风险雷达/近期信号/投研结论），禁止自行更改
- [ ] 评分体系使用 1-10 分制，禁止使用星级制或 5 分制
- [ ] 基础档案表包含「数据来源」列（三列结构：项目/详情/数据来源）
- [ ] 所有表格格式规范统一，数据均标注来源
- [ ] 可读性规范已遵循：合理使用列表和表格呈现信息
- [ ] 风险章节已标注风险严重程度等级
- [ ] 附录部分包含完整的数据来源索引
- [ ] 已成功创建钉钉在线文档并返回访问链接

## 典型问题排查与应对

| 典型问题 | 根因 | 应对策略 |
|---------|------|---------|
| 基础信息缺失 | 仅执行单轮搜索，遗漏创始人、融资、上市等关键维度 | 严格执行 A1 三轮搜索方案，每轮聚焦不同信息维度 |
| 业务架构不清晰 | 未获取营收占比数据，仅停留在定性描述层面 | 必须从财报或权威报告中提取量化数据，无数据时标注"待补充" |
| 竞品分析失真 | 对比维度不统一，或选择对比对象缺乏代表性 | 采用统一对比表格框架，选取真正的主要竞品（行业前 5） |
| 财务数据来源单一 | 仅引用单一数据源，存在信息偏差风险 | 至少通过 2 个独立来源交叉验证，差异超过 10% 需特别说明 |
| 财务数据口径不一致 | 不同指标取自不同财年，统计口径存在差异 | 统一归集至同一财年，明确标注每个数据的具体年份 |
| SWOT 分析空洞 | 优劣势描述过于笼统，缺乏具体事实佐证 | 每条 SWOT 要点必须附加"依据：{具体来源}"标注 |
| 风险评估覆盖不全 | 仅关注市场和竞争风险，忽略监管、技术、运营等其他风险类别 | 严格按照 8 类风险逐一评估，无显著风险时标注"当前无重大风险" |
| 动态信息滞后 | 仅检索到半年前的历史新闻 | 搜索关键词添加最新年份限定，如"2024 2025"，直接抓取新闻页面获取最新资讯 |
| 结论缺乏支撑 | 给出判断结论但未提供数据依据 | 每个结论必须附带"依据"字段，推测性内容需标注"需进一步验证" |
| draft.md 结构混乱 | 各模块写入格式缺乏统一标准 | 严格遵循 SKILL 文档中每个子模块的写入模板规范 |
| 报告篇幅失控 | 堆砌过多细节信息，核心结论被淹没 | 标准版本控制在 10-20 页，确保摘要部分可在 1 页内读完 |
| 表格格式不统一 | 不同章节表格风格差异较大 | Writer Agent 统一格式规范：表头加粗、数值右对齐、金额统一单位 |
| 公司主体混淆 | 母子公司、关联公司信息混杂 | 任务分解第一步"明确公司主体范围"必须清晰界定研究边界 |
| 钉钉文档写入异常 | markdown 内容超长或格式存在错误 | 采用分段写入策略（先 overwrite 后 append），确保换行符为真实换行符 |
| 市场份额加总超 100% | 引用了不同口径或不同来源的份额数据混用 | 统一数据口径和来源，同一表格内份额之和须 ≤100%，不同口径须注明 |
| 报告章节名与模板不一致 | Writer Agent 未读取或未遵循 report-template.md | 阶段 4 校验步骤须逐章节核对模板章节名，不一致则修正后再写入 |
| 非上市公司章节缺失 | 融资/上市章节因无数据被直接跳过 | 须显式说明"该公司为XX全资子公司，无独立融资记录"，不得留空 |
