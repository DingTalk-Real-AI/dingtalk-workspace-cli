# 数据获取详细流程

> 本文件描述年度文档数据获取的完整流程，包括分页策略、数据筛选和异常处理。

## 获取流程总览

```
Step 1: 获取根目录文档列表
    ↓
Step 2: 递归遍历子文件夹
    ↓
Step 3: 逐文档获取元信息
    ↓
Step 4: 按年度范围筛选
    ↓
Step 5: 汇总文档清单
```

---

## Step 1: 获取根目录文档列表

### 命令

```bash
dws doc list --format json
```

### 返回结构

```json
{
  "nodes": [
    {
      "nodeId": "xxx",
      "name": "文档标题",
      "type": "file",
      "url": "https://alidocs.dingtalk.com/i/nodes/xxx"
    }
  ],
  "nextPageToken": "xxx"
}
```

### 处理逻辑
1. 解析返回的 `nodes` 数组
2. 将 `type=file` 的节点加入文档列表
3. 将 `type=folder` 的节点加入待遍历文件夹队列
4. 若 `nextPageToken` 非空，继续翻页

---

## Step 2: 递归遍历子文件夹

### 命令

```bash
dws doc list --folder <FOLDER_ID> --page-size 50 --format json
```

### 遍历策略
- 使用广度优先遍历（BFS），避免递归过深
- 每个文件夹最多翻页 10 次（即最多 500 个子项）
- 遇到嵌套文件夹继续加入队列

### 分页处理

```
初始请求: dws doc list --folder <ID> --format json
    ↓ 检查 nextPageToken
有值 → dws doc list --folder <ID> --page-token <TOKEN> --format json
    ↓ 重复直到 nextPageToken 为空或达到翻页上限
```

### 防护措施
- 文件夹深度上限: 5 层（超过则跳过并记录警告）
- 单文件夹翻页上限: 10 次
- 总文档数上限: 2000 篇（超过则停止遍历，使用已获取数据）

---

## Step 3: 逐文档获取元信息

### 命令

```bash
dws doc info --node <NODE_ID> --format json
```

### 返回结构

```json
{
  "nodeId": "xxx",
  "name": "文档标题",
  "type": "alidoc",
  "creatorName": "张三",
  "createTime": "2024-03-15T10:30:00+08:00",
  "modifiedTime": "2024-06-20T14:22:00+08:00",
  "url": "https://alidocs.dingtalk.com/i/nodes/xxx"
}
```

### 提取字段

| 字段 | 用途 |
|------|------|
| `name` | 文档标题，用于主题分类和榜单展示 |
| `type` | 文档类型，用于类型分布统计 |
| `creatorName` | 作者，用于筛选（若指定作者） |
| `createTime` | 创建时间，用于年度范围筛选和趋势分析 |
| `modifiedTime` | 最后更新时间，用于活跃度分析 |
| `url` | 文档链接，用于报告中的超链接 |

### 批量处理策略
- 文档数 <= 50：逐一获取
- 文档数 > 50：每获取 20 篇暂停 1 秒，避免请求过快
- 某篇文档获取失败：记录错误，跳过继续处理其余文档

---

## Step 4: 按年度范围筛选

### 筛选规则

```
保留条件: createTime >= {年度开始} AND createTime <= {年度结束}
```

- 年度开始: `{YYYY}-01-01T00:00:00+08:00`
- 年度结束: 若为当前年度且未到年底，使用今天的日期 `{YYYY}-{MM}-{DD}T23:59:59+08:00`；否则使用 `{YYYY}-12-31T23:59:59+08:00`
- 时区: 统一使用 `+08:00`（北京时间）

### 排除本次报告文档
- 在阶段 4（生成报告文档）中创建的报告文档，其 `nodeId` 必须从统计数据中排除
- 实现方式: 数据获取在报告创建之前完成，因此正常流程下不会包含本次报告；但若用户重新生成（报告文档已存在），需按标题模式（如包含"文档创作年度报告"）排除匹配的文档

### 额外筛选（可选）
- 若用户指定作者: `creatorName == {指定作者}`
- 若用户指定文档类型: `type == {指定类型}`

### 去重
- 按 `nodeId` 去重，同一文档只保留一条记录

---

## Step 5: 汇总文档清单

### 输出格式

将筛选后的文档整理为结构化清单，供阶段 3（数据分析）使用：

```
文档清单:
- 文档总数: XX 篇
- 类型分布: 文档 XX / 表格 XX / 幻灯片 XX / 其他 XX
- 时间跨度: YYYY-MM-DD ~ YYYY-MM-DD

逐文档明细:
| 序号 | 标题 | 类型 | 创建时间 | 最后更新 | 链接 |
|------|------|------|---------|---------|------|
| 1 | ... | ... | ... | ... | ... |
```

---

## 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| `dws doc list` 返回空 | 检查是否有子文件夹，若全部为空则终止并告知用户 |
| `dws doc info` 单篇失败 | 跳过该文档，在最终报告中标注"部分文档信息获取失败" |
| 分页 token 无效 | 停止该文件夹的翻页，使用已获取数据 |
| 文档总数超过 2000 | 停止遍历，标注"数据可能不完整，仅展示前 2000 篇" |
| 网络超时 | 重试 1 次，仍失败则跳过 |
| 年度范围内无文档 | 终止流程，告知用户"未找到该年度的文档记录" |

---

## 数据质量说明

由于 `dws doc` 命令目前不直接提供 PV、UV、点赞、评论等互动指标，报告中的排序和分析将基于以下可用数据：

- **文档标题和类型**: 用于主题分类
- **创建时间**: 用于产出趋势分析
- **最后更新时间**: 用于活跃度分析（更新频次 = 最后更新 - 创建时间）
- **文档链接**: 用于报告中的超链接引用

若未来 `dws doc` 支持互动指标，可扩展数据获取流程以包含这些字段。
