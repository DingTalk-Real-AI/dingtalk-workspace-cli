# 文档创作年度报告

基于钉钉文档数据，生成包含产出统计、Top 榜单、主题分布、趋势变化和亮点提炼的年度总结报告，最终产出为钉钉在线文档。

## 涉及产品

| 产品 | 用途 | 对应命令 |
|------|------|---------|
| 文档 (doc) | 搜索文档列表、获取文档元信息、创建年度报告文档 | `dws doc search / list / info / create / update` |

## 前置依赖

- 需要 `dws`（dingtalk-workspace CLI）已安装并登录

## 能力清单

| 动作 | 命令 | 必填参数 | 安全等级 | 说明 |
|------|------|----------|---------|------|
| 搜索文档 | `dws doc search` | --query | 只读 | 按关键词搜索，不传则返回最近访问 |
| 浏览文档列表 | `dws doc list` | -- | 只读 | 浏览文件夹下的文档列表 |
| 获取文档元信息 | `dws doc info` | --node | 只读 | 获取标题、类型、权限等元信息 |
| 创建文档 | `dws doc create` | --name | 写入 | 创建年度报告文档 |
| 写入/追加内容 | `dws doc update` | --node, --markdown, --mode | 写入 | 向报告文档写入 Markdown 内容 |

## 操作指引

### 数据获取
1. MUST: 所有 `dws` 命令必须加 `--format json` 以获取可解析输出
2. MUST: 确认年度范围（默认自然年 1月1日~12月31日），用户可自定义
3. SHOULD: 文档量较大时按文件夹分批遍历，每批使用 `--page-size 50` 分页
4. NOTE: `dws doc search` 不传 `--query` 时返回最近访问文档，需配合 `dws doc list` 遍历完整列表
5. TRAP: `dws doc info` 返回的是元信息（标题、类型、创建时间、更新时间），不包含 PV/UV 等互动指标

### 报告生成
1. MUST: 报告必须以钉钉在线文档形式交付，不得以对话文本替代
2. MUST: 使用 `dws doc create --name "<报告标题>" --format json` 创建文档，提取返回的 `nodeId`
3. SHOULD: 内容较长时分段写入——先 `--mode overwrite` 写入第一部分，后续用 `--mode append` 追加
4. NOTE: `--markdown` 参数中的换行必须使用真实换行符（Unicode U+000A），不是字面量 `\n`
5. TRAP: `--mode overwrite` 会清空原内容，追加内容务必用 `--mode append`

### 报告内容
1. MUST: 严格按照 [references/report-template.md](./references/report-template.md) 的章节结构、章节名称和格式生成，不得自行更改章节名称或格式
2. MUST: 报告的七大章节名称固定为：**全年总览 → 代表作品 → 创作方向 → 创作节奏 → 年度标记 → 创作画像 → 下一步**，禁止使用其他名称（如"Top 10 文档""主题分布""展望"等）
3. MUST: 所有数据必须来自实际获取的文档信息，禁止编造
4. MUST: 本次生成的报告文档本身不得出现在 Top 榜单和统计数据中——在数据获取阶段排除它
5. MUST: 每个章节的表格列数和列名必须与模板一致，不得省略列或自行增减
6. SHOULD: 数据指标使用表格展示，并列项 >=3 使用列表
7. TRAP: `dws doc` 不提供 PV/UV/点赞/评论等互动指标，排序基于更新频次和创建时间，不得编造互动数据
8. TRAP: 简化模式（<30 篇）是合并章节而非删除章节——"创作画像"的数据合并到"全年总览"中展示，"创作节奏"仍需保留

## 工作流

### 阶段 1: 确认参数
1. 从用户话中提取年度范围（默认当前年度 1月1日 至今天）
2. 确认筛选条件：作者（默认当前用户）、文档类型（可选）
3. 若当前日期未到年底，统计范围自动调整为 1月1日 ~ 今天，报告标题和正文中标注实际统计截止日期
4. 校验: 年度范围合法
5. 失败: 年度范围无法解析 → 询问用户明确时间

### 阶段 2: 获取文档列表
依赖: 阶段 1 的年度范围和筛选条件
1. 调用 `dws doc list --format json` 获取文档根目录列表
2. 递归遍历子文件夹: `dws doc list --folder <FOLDER_ID> --format json`
3. 对每篇文档调用 `dws doc info --node <NODE_ID> --format json` 获取元信息
4. 按创建时间筛选出年度范围内的文档
5. 校验: 文档列表非空
6. 失败: 列表为空 → 告知用户"未找到该年度的文档"，终止流程

详细获取流程见 [references/data-collection.md](./references/data-collection.md)

### 阶段 3: 数据分析
依赖: 阶段 2 的完整文档列表和元信息
1. 按 [references/report-template.md](./references/report-template.md) 的七大章节逐一分析，每个章节的格式和表格列名严格遵循模板中的强制约束
2. 数据量判断: 文档数 < 30 篇时启用简化模式——将"创作画像"的数据行合并到"全年总览"的核心数字表格中，"下一步"精简为 2 条建议，其余章节保留
7. 校验: 各分析模块数据完整
8. 失败: 某模块数据不足 → 标注"数据不足，仅供参考"

### 阶段 4: 生成报告文档
依赖: 阶段 3 的分析结果
1. 调用 `dws doc create --name "{年度}年文档创作年度报告" --format json`
2. 提取返回的 `nodeId` 和文档 URL
3. 按 [references/report-template.md](./references/report-template.md) 模板组织 Markdown 内容
4. 调用 `dws doc update --node <NODE_ID> --markdown "<报告内容>" --mode overwrite --format json`
5. 内容过长时分段: 先 overwrite 第一部分，后续 append 追加
6. 校验: update 命令返回成功
7. 失败: 写入失败 → 重试 1 次，仍失败则将报告内容保存为本地 Markdown 文件并告知用户

### 阶段 5: 交付
依赖: 阶段 4 的文档 URL
1. 向用户展示报告概要（一段话，150 字以内）
2. 附上钉钉在线文档链接
3. 禁止将报告全文输出到对话中

## 使用示例

### 示例 1: 生成当年报告
**用户说**: "帮我生成年度文档报告"
**执行步骤**:
1. 确认年度范围为当前年度（如 2025-01-01 ~ 2025-12-31）
2. 调用 `dws doc list --format json` 获取文档列表
3. 逐文档调用 `dws doc info --node <ID> --format json` 获取元信息
4. 按创建时间筛选年度内文档，进行多维度分析
5. 调用 `dws doc create --name "2025年文档创作年度报告" --format json`
6. 调用 `dws doc update --node <NODE_ID> --markdown "<报告内容>" --mode overwrite --format json`
**期望输出**: "已生成 2025 年文档创作年度报告，全年共创作 XX 篇文档。[查看报告](文档URL)"

### 示例 2: 指定年份
**用户说**: "统计一下我 2024 年的文档创作情况"
**执行步骤**:
1. 确认年度范围为 2024-01-01 ~ 2024-12-31
2. 同示例 1 的步骤 2-6，筛选条件改为 2024 年
**期望输出**: "已生成 2024 年文档创作年度报告。[查看报告](文档URL)"

### 示例 3: 口语化触发
**用户说**: "我今年写了哪些文档？帮我做个回顾"
**执行步骤**:
1. 识别意图为年度文档回顾，确认年度范围为当前年度
2. 同示例 1 的步骤 2-6
**期望输出**: "已生成年度文档创作报告，包含产出统计、Top 榜单和趋势分析。[查看报告](文档URL)"

### 示例 4: 最简触发
**用户说**: "年度报告"
**执行步骤**:
1. 使用默认参数：当前年度、当前用户、所有文档类型
2. 同示例 1 的步骤 2-6
**期望输出**: "已生成 2026 年文档创作年度报告。[查看报告](文档URL)"

## 错误处理

| 阶段 | 错误 | 处理 | 是否终止流程 |
|------|------|------|-------------|
| 阶段 2 | `dws doc list` 返回空 | 告知用户"未找到文档" | 是 |
| 阶段 2 | `dws doc info` 某文档获取失败 | 跳过该文档，继续处理其余 | 否 |
| 阶段 2 | 分页遍历超时 | 使用已获取的部分数据继续分析，标注"数据可能不完整" | 否 |
| 阶段 4 | `dws doc create` 失败 | 重试 1 次，仍失败则报告错误信息 | 是 |
| 阶段 4 | `dws doc update` 内容过长失败 | 分段写入（先 overwrite 第一部分，后续 append） | 否，回退到分段写入 |
| 阶段 4 | `dws doc update` 权限不足 | 提示用户检查文档权限 | 是 |
| 全局 | `dws` 命令不可用（未安装或未登录） | 提示用户确认 dws 已安装，并重新执行上一条命令（最多重试两次） | 是 |
