# wukong-doc-marketing-campaign-plan

用于生成营销活动或传播方案。主文件聚焦触发、核心结构和最短工作流；详细模板、人群分析和预算方法见 `references/`。

## 严格禁止 (NEVER DO)

- 不要在没有目标和目标人群时直接写完整方案。
- 不要编造预算、ROI 或投放数据。
- 不要把渠道清单堆满而没有优先级和节奏。
- 不要忽略风险预案，尤其是预算、产能和渠道风险。
- 不要用空泛口号代替卖点和传播主张。
- 不要把最终交付停留在聊天回复，必须落到钉钉文档。

## Tool 总览

| 工具 | 用途 | 安全等级 | 必填参数 | 示例 |
|---|---|---|---|---|
| `dws doc search/read` | 读取历史活动和品牌资料 | `L1` | `keyword` 或 `node` | 搜索“营销方案” |
| `knowledge_answer` | 查询品牌和内部经验 | `L1` | `query_list` | 查品牌定位 |
| `web_search` | 查询人群、竞品、渠道信息 | `L1` | `query` | 查竞品案例 |
| `fetch` | 深挖外部案例详情 | `L1` | `url` | 获取案例正文 |
| `dws doc create/update` | 生成最终钉钉文档 | `L2` | `name` 或 `node` | 创建方案文档 |

## 意图判断决策树

用户提到“活动营销方案/推广计划/活动策划”：
- 有明确活动目标和人群 → 直接生成方案
- 缺目标或人群 → 先补最小输入

用户提到“新品发布/节日大促/品牌活动”：
- 默认输出传播节奏、渠道矩阵和预算框架

用户提到“投放怎么做/达人怎么配/什么渠道有效”：
- 重点展开渠道与预算章节

| 单条社媒帖子或内容日历 | `social-content` |

## 核心工作流

1. 确认最小输入：
   - 活动目标
   - 目标人群
   - 时间周期
   - 预算范围
2. 收集资料：
   - 内部历史活动
   - 品牌卖点
   - 人群洞察
   - 竞品案例
3. 生成方案结构：
   - 目标与 KPI
   - 人群洞察
   - 核心主张
   - 渠道矩阵
   - 节奏规划
   - 预算与风险
4. 交付钉钉文档：

```text
-> dws doc search --query "618 活动营销方案"
<- 返回历史活动文档

-> web_search "美妆 618 人群 触媒习惯 2025"
<- 返回外部洞察

-> fetch CASE_URL
<- 提取竞品活动动作和结果

-> dws doc create --name "618 活动营销方案"
<- 返回文档节点
```

## 上下文传递规则

| 操作 | 从返回中提取 | 用于 |
|---|---|---|
| 历史方案读取 | `目标`, `渠道`, `预算`, `效果` | 历史经验复用 |
| 人群洞察 | `画像`, `偏好`, `触媒渠道` | 人群章节 |
| 竞品案例 | `玩法`, `卖点`, `渠道动作` | 竞品分析与策略选择 |
| 文档创建 | `node` 或 `url` | 最终文档更新 |

## MCP 调用约束

- 单次调用优先收集一个维度，例如人群、竞品或渠道。
- 参数尽量不超过 5 个，嵌套不超过 1 层。
- 预算和 ROI 只能基于用户输入或可核验资料，不能在写入阶段补造。

## 错误处理

- 用户不给预算时，只能给预算框架，不能填写具体数字。
- 用户不给时间周期时，只能给阶段模板，不能写具体排期。
- 竞品信息不足时，先做“品牌内策略版”，同时标记缺失项。
- 目标太宽泛时，先收敛成曝光、获客、转化三类之一。

## 质量标准

- 目标和 KPI 必须可量化或可验证。
- 渠道矩阵必须有优先级，不能平均用力。
- 预算只能写有依据的数字或明确写成框架。
- 最终交付必须是钉钉云文档。

## Checklist

- 是否明确活动目标、目标人群、时间周期
- 是否给出核心卖点和传播主张
- 是否区分渠道优先级
- 是否预算有依据或明确为框架
- 是否已创建并更新钉钉文档
