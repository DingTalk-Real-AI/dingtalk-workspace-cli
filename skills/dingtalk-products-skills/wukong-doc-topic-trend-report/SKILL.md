# wukong-doc-topic-trend-report

用于生成带时间窗和来源标注的主题动态报告。主文件只保留时间解析、检索、去重和交付主线；分类框架、样例和检查项见 `references/`。

## 严格禁止 (NEVER DO)

- 不要在未确认时间范围前开始汇总动态。
- 不要把窗口外内容混入报告。
- 不要把转载、评论或二手解读放在官方源之前。
- 不要同一事件重复收录多条。
- 不要无法确认发布时间还强行纳入。
- 不要把最终交付停留在会话消息里，必须落到钉钉文档。

## Tool 总览

| 工具 | 用途 | 安全等级 | 必填参数 | 示例 |
|---|---|---|---|---|
| `web_search` | 多源检索候选动态 | `L1` | `query` | 搜某主题近一周动态 |
| `fetch` | 核验发布时间和内容 | `L1` | `url` | 抓页面详情 |
| `dws doc create/update` | 输出最终钉钉文档 | `L2` | `name` 或 `node` | 创建周报文档 |

## 意图判断决策树

用户提到“日报/周报/动态汇总/趋势追踪”：
- 指定主题和时间范围 → 进入动态报告流程
- 缺时间范围 → 默认今天，并提示当前窗口

用户提到“最近一周/上个月/某个日期范围”：
- 先转成明确的开始和结束时间

用户提到“某主题最新动态”：
- 默认按权威源优先，并做去重

| 如果用户要… | 应使用 |

| 管理层季度或年度经营复盘 | `wukong-doc-business-review-report` |

## 核心工作流

1. 解析输入：
   - 主题
   - 时间窗口
   - 可选子领域或关注公司
2. 检索候选：
   - 官方源
   - 权威媒体
   - 必要时补充社区来源
3. 核验与去重：
   - 核对发布时间
   - 剔除窗口外内容
   - 同事件合并
4. 分类后输出钉钉文档：

```text
-> web_search "AI Agent 最近一周 发布 更新 公告"
<- 返回候选链接

-> fetch LINK
<- 提取发布时间、标题、核心内容

-> dws doc create --name "AI Agent 周动态"
<- 返回文档节点
```

## 上下文传递规则

| 操作 | 从返回中提取 | 用于 |
|---|---|---|
| 搜索候选 | `title`, `url`, `source` | 候选池 |
| 详情核验 | `published_at`, `summary`, `keywords` | 时间筛选和分类 |
| 去重整理 | `event_key`, `primary_source` | 合并同事件 |
| 文档创建 | `node` 或 `url` | 最终文档更新 |

## MCP 调用约束

- 检索和核验分开做，先搜候选，再逐条确认时间。
- 参数尽量不超过 5 个，嵌套不超过 1 层。
- 先完成时间核验和去重，再写入钉钉文档。

## 错误处理

- 无法确认发布时间的条目直接剔除。
- 同一事件出现多个来源时，保留官方或原始来源。
- 用户主题过宽时，优先给官方发布、产品更新、行业市场三类。
- 用户只说“今天”或“昨天”时，按用户时区解析时间窗。

## 质量标准

- 每条内容都必须有发布时间和来源。
- 所有条目必须落在指定时间窗口内。
- 同一事件只保留一个主条目。
- 最终交付必须是钉钉云文档。

## Checklist

- 是否时间范围明确
- 是否每条都有发布时间和来源
- 是否剔除了窗口外内容
- 是否完成去重和分类
- 是否已创建并更新钉钉文档
