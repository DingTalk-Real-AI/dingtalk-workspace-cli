# 钉钉论文深度解读 Skill

对学术论文或研究话题进行严格学术分析，生成结构化钉钉在线文档，覆盖六个维度：研究概览 → 核心论文解构 → 研究脉络 → 前沿进展对比 → 批判性评估 → 开放问题与研究方向。

## 适用边界

- **Distinct from**：`wukong-doc-competitive-analysis`（商业竞品对比）、`literature-survey`（大规模文献计量综述，无核心论文精读需求）
- **Do NOT use for**：非学术内容的产品分析、市场报告、商业竞品对比，或仅需摘要的博客/文章总结

## 工具权限

| 工具 | 用途 | 约束 |
|------|------|------|
| `web_search` | 论文检索、引用追踪、批判性视角搜索 | 仅访问公开学术资源 |
| `WebFetch` | 获取论文 PDF / HTML 全文 | 必须阅读原文，禁止仅凭摘要下结论 |
| `dws doc create` | 新建钉钉文档 | 每次任务只新建一个文档 |
| `dws doc update` | 写入报告内容 | 仅操作本次 create 返回的 nodeId |

所有 `dws` 命令必须附加 `--format json`。

## 触发意图识别

| 用户表达 | 分析重点 | 执行路径 |
|---------|---------|---------|
| "帮我精读这篇论文" / "分析一下这篇 paper" | 核心论文深度解构 | Step 1-2 为主，Step 3-4 补充背景 |
| "对 {topic} 做 deep research" | 六维度全量分析 | Step 1-6 全流程 |
| "总结最新的 {领域} 研究进展" | 前沿进展对比为主 | Step 3-4 为主，Step 5 批判性评估 |
| "这篇论文和近期工作有什么关系" | 前沿进展 + 关系论证 | Step 1-2 + Step 4 |
| "帮我找这个领域的研究空白" | 批判性评估 + 开放问题 | Step 5 + Step 6 |
| "这篇论文的定理证明是什么" | 理论保证深度解析 | Step 2（§1.3 理论保证） |

## 易混淆场景路由

| 用户说 | 判断线索 | 正确路由 |
|--------|---------|---------|
| "帮我写文献综述" | 强调引用计量、分类统计、大规模文献梳理，无核心论文精读需求 | 路由到 literature-survey |
| "帮我做竞品分析" | 关注商业产品对比、定价、市场份额 | 路由到 wukong-doc-competitive-analysis |
| "总结一下这篇博客/文章" | 非学术内容，无论文结构 | 普通摘要任务，非本 Skill |
| "帮我写一篇论文" | 创作需求，非解读需求 | 学术写作 Skill，非本 Skill |

## 典型场景示例

**示例 1：单论文精读**
- 用户：帮我精读 Attention Is All You Need 这篇论文
- 执行：Step 1 获取 arXiv PDF → Step 2 完成核心论文解构（问题建模/方法推导/理论保证/算法/实验可信度）→ Step 3 建立 NLP 注意力机制研究脉络 → Step 4 调研近 24 个月前沿进展并系统对比 → Step 5 批判性评估（局限性 + 争议点）→ Step 6 创建钉钉文档
- 输出：含 LaTeX 公式、横向对比矩阵、关系论证、批判性评估、开放问题的完整钉钉文档链接

**示例 2：话题 Deep Research**
- 用户：对 diffusion model 做 deep research，重点看近两年进展
- 执行：确认话题关键词和时间窗口 → Step 3 建立扩散模型领域演进时间线 → Step 4 搜索近 24 个月顶会论文，建立研究分支分类 + 横向对比矩阵 → Step 5 批判性评估当前研究格局 → Step 6 创建钉钉文档
- 输出：含研究脉络、分支分类、横向对比矩阵、批判性评估、潜在突破口的话题综述钉钉文档链接

## 输入参数

| 参数 | 是否必填 | 说明 |
|------|---------|------|
| 论文标题 / arXiv ID / DOI / 话题描述 | 必填 | 分析的起点 |
| 近期工作时间窗口 | 可选 | 默认近 24-36 个月 |
| 重点分析方向 | 可选 | 理论 / 实验 / 应用，默认全覆盖 |
| 报告长度 | 可选 | 简洁版（5-8 页）/ 标准版（15-20 页）/ 详细版（25-30 页） |

## 执行流程

### Step 1：定位并获取核心论文

搜索策略和命令见 [search-patterns](references/search-patterns.md)（Step 1 部分）。

必须阅读：摘要 + 引言 + 方法 + 主要定理/关键引理 + 实验 + 局限性

记录：标题 / 作者 / 会议或期刊 / 年份 / arXiv 版本号（v1/v2/...）

**步骤信息流**

| 步骤 | 产出 | 流向 |
|------|------|------|
| Step 1 | 论文全文、元数据、版本信息 | → Step 2 结构化分析 |
| Step 2 | 核心论文六维度分析结果、统一符号体系 | → Step 4 近期工作搜索关键词；→ Step 6 报告 §1 |
| Step 3 | 领域演进时间线、canonical 问题形式 | → Step 4 搜索关键词扩展；→ Step 6 报告 §2 |
| Step 4 | 结构化卡片列表、研究分支分类 | → Step 5 批判性评估；→ Step 6 报告 §3 |
| Step 5 | 局限性分析、争议点、可信度评级 | → Step 6 报告 §4 |
| Step 6a | nodeId | → Step 6b dws doc update |

### Step 2：核心论文结构化分析

对应报告 §1，详细分析方法见 [search-patterns](references/search-patterns.md)（Step 2 部分）：

- **§1.1 问题建模**：canonical objective（LaTeX）+ 符号表 + 关键假设（含"能否放松"）
- **§1.2 方法与推导**：完整推导路线 + 核心公式逐项解释
- **§1.3 理论保证**：每个定理严格按"前提条件 → 结论 → 实践意义 → 证明要点"四段式
- **§1.4 算法实现**：迭代更新式 + 复杂度 + 实现注意事项
- **§1.5 实验可信度评估**：5 个维度 ⭐ 评级（不得默认满星）+ 综合判断

### Step 3：研究脉络调研

对应报告 §2，搜索命令见 [search-patterns](references/search-patterns.md)（Step 3 部分）。

产出：领域演进时间线（早期/中期/近期）+ 统一符号下的 canonical 问题形式 + 主流研究范式

### Step 4：前沿进展调研

对应报告 §3，搜索策略见 [search-patterns](references/search-patterns.md)（Step 4 部分）。

时间窗口：默认近 24-36 个月，超出范围的论文仅在 §2 研究脉络中引用。

产出：研究分支分类表 + 横向对比矩阵 + 每篇论文的进展关系论证（须含技术细节，禁止仅罗列摘要）

### Step 5：批判性评估

对应报告 §4，独立于论文作者立场进行批判，见 [search-patterns](references/search-patterns.md)（Step 5.4 部分）。

产出：
- 核心论文局限性（理论/实验/工程/适用范围，每条须具体可验证）
- 领域争议点（若存在）
- 可信度综合评级（4 个维度，不得默认满星）

> 此步骤是本 Skill 区别于其他论文解读工具的核心差异点，不得跳过。

### Step 6：创建钉钉文档

**Step 6a**：创建文档

```bash
dws doc create --name "{论文标题 / Topic} 深度解读 {YYYY-MM}" --format json
```

从返回结果中提取 `nodeId`。

**Step 6b**：写入报告（必须使用文件路径方式，禁止内联 markdown）

```bash
# 先将报告内容写入临时文件
# 报告结构见 output-template.md
dws doc update --node <nodeId> --markdown-file /tmp/paper-report.md --mode overwrite --format json
```

返回文档链接给用户，并附 2-3 条核心发现摘要（含最重要的定理结论或实验发现）。

## 错误处理

| 错误场景 | 处理方式 |
|---------|---------|
| PDF 获取失败（超时 / 403） | 依次尝试：arXiv HTML 版（ar5iv.org）→ Semantic Scholar → ACL Anthology → OpenReview；仍失败则标注「原文获取失败，分析基于摘要，关键定理可信度受限」 |
| 论文不存在 / 标题有误 | 告知用户，请求提供正确的 arXiv ID 或 DOI，**不得继续分析** |
| 近期工作搜索无结果 | 扩大时间窗口至 48 个月，或换用英文关键词 / 同义词重试；仍无结果则标注「近期工作数据不足，对比分析受限」 |
| `dws doc create` 失败 | 重试一次；仍失败则将完整报告内容输出到对话中，提示用户手动创建文档 |
| `dws doc update` 失败 | 检查 nodeId 是否正确，重试 `--mode overwrite`；仍失败则分段输出报告内容 |

## NEVER DO

- **禁止仅凭摘要或博客二手解读下结论**：必须获取并阅读原文 PDF
- **禁止编造引用**：所有引用必须含作者 + 完整标题 + 会议/期刊 + 年份，不得伪造
- **禁止跳过批判性评估**（§4）：这是本 Skill 的核心差异点，不得以"时间不足"为由省略
- **禁止使用超出时间窗口的近期工作**：超出 24-36 个月的论文仅在研究脉络（§2）中引用
- **禁止输出未达质量门禁的报告**：质量门禁 7 项须全部通过，见 quality-checklist.md
- **禁止修改已有文档**：只能写入本次 `create` 返回的 nodeId 对应文档

## 报告质量门禁

> 以下 7 项须全部满足，才可执行 `dws doc create`。详细检查清单见 [quality-checklist](references/quality-checklist.md)。

| 维度 | 最低要求 |
|------|---------|
| 公式覆盖 | 至少 3 个关键公式（canonical form / 核心方法 / 定理） |
| 推导深度 | 至少 1 条完整推导路线（proof sketch 可接受） |
| 对比广度 | 至少 1 张横向对比矩阵（维度统一，横向可比） |
| 论证质量 | 至少 3 条关系论证句（含技术细节，非摘要罗列） |
| 批判深度 | 至少 1 条具体可验证的局限性（非空泛表述） |
| 结论可验证性 | 无空泛形容词，所有结论可指向公式 / 实验数据 |
| 开放问题质量 | 至少 3 条可验证的后续研究问题 |

## 常见失败模式与修复

| 失败表现 | 根因 | 正确做法 |
|---------|------|---------|
| 报告停留在摘要级，缺少公式和定理细节 | 未获取原文 PDF 或未完整阅读 | 回到 Step 1，必须阅读 Method + Theorems + Experiments 章节 |
| 近期工作仅罗列，无分类无论证 | 跳过研究分支分类，直接堆砌摘要 | 先建立分支分类框架，再写横向对比矩阵，最后写关系论证句 |
| 公式出现但无解释，读者无法理解 | 符号未定义，缺少直觉解释 | 每个公式逐项解释含义，每个符号首次出现时定义 |
| 定理结论写了但假设缺失 | 跳过前提条件直接写结论 | 严格使用"前提条件 → 结论 → 实践意义 → 证明要点"四段式 |
| 不同章节符号互相冲突 | 未在开头建立统一符号体系 | Step 2 §1.1 建立符号表，后续所有章节严格遵循 |
| 批判性评估（§4）缺失或空洞 | 将局限性一笔带过，未独立成章 | §4 必须独立成章，局限性须具体可验证，可信度评级须有理由 |
| 近期工作引用了 5 年前的论文 | 未严格执行时间窗口 | 严格限定 24-36 个月，超出范围的论文仅在 §2 研究脉络中引用 |
| 文档创建失败后直接终止任务 | 未做降级处理 | 重试一次；仍失败则将报告内容输出到对话中 |
