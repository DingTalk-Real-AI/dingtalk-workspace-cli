# wukong-doc-industry-research

用于生成行业研究报告。主文件负责触发判断、工具选择和最短研究流程；详细检索策略、分析框架和示例见 `references/`。

## 严格禁止 (NEVER DO)

- 不要把零散搜索结果直接拼接成报告。
- 不要用单一来源给出关键市场规模或增长结论。
- 不要在用户指定时间范围后混入窗口外资料。
- 不要只列公司名字而不分析竞争关系。
- 不要编造数据、链接、机构观点或政策内容。
- 不要忽略来源年份和发布时间。

## Tool 总览

| 工具 | 用途 | 安全等级 | 必填参数 | 示例 |
|---|---|---|---|---|
| `web_search` | 广度检索公开资料 | `L1` | `query` | 查行业规模 |
| `fetch` | 获取网页/报告详情 | `L1` | `url` | 抓报告正文 |
| `dws doc search/read` | 查询内部已有研究 | `L1` | `keyword` 或 `node` | 查历史报告 |
| `knowledge_answer` | 补内部知识 | `L1` | `query_list` | 查组织内部结论 |
| `dws doc create/update` | 生成最终钉钉文档 | `L2` | `name` 或 `node` | 创建研究报告 |

## 意图判断决策树

用户提到“行业研究/赛道分析/市场分析”：
- 关注全面分析 → 输出标准研究报告
- 只关心一个维度 → 输出专题分析

用户提到“市场规模/竞争格局/技术趋势/政策环境”：
- 作为重点章节处理，但仍保留最小背景说明

用户提到“帮我判断要不要进入这个市场”：
- 默认增加机会、风险和进入建议

## 核心工作流

1. 明确输入：
   - 行业或赛道
   - 地理范围
   - 时间范围
   - 关注重点
2. 广度检索：
   - 定义与边界
   - 市场规模
   - 竞争格局
   - 技术与政策
   - 投融资动态
3. 深度分析：
   - 重点公司
   - 五力或 SWOT
   - 趋势与机会风险
4. 生成钉钉文档：

```text
-> web_search "AI Agent 行业 市场规模 2025"
<- 返回候选报告

-> fetch REPORT_URL
<- 提取市场规模、年份、来源

-> dws doc search --query "AI Agent 行业研究"
<- 返回内部历史文档

-> dws doc create --name "AI Agent 行业研究报告"
<- 返回文档节点
```

## 上下文传递规则

| 操作 | 从返回中提取 | 用于 |
|---|---|---|
| 广度检索 | `来源`, `年份`, `关键数据` | 市场与趋势章节 |
| 公司分析 | `公司`, `份额`, `优势`, `动向` | 竞争格局章节 |
| 政策检索 | `政策名称`, `日期`, `影响` | 政策环境章节 |
| 文档创建 | `node` 或 `url` | 最终文档更新 |

## MCP 调用约束

- 单次搜索聚焦一个研究维度，不要把多个维度混在一个查询里。
- 参数尽量不超过 5 个，嵌套不超过 1 层。
- 先完成检索和核验，再创建或更新钉钉文档。

## 错误处理

- 核心数据至少用两类独立来源交叉验证。
- 数据差异明显时，保留两者并解释口径差异。
- 找不到权威来源时，只能输出“可验证信息摘要”，不能补造结论。
- 用户需求很宽时，先按“定义、规模、竞争、趋势、机会风险”五段落交付。

## 质量标准

- 核心数据至少两类独立来源交叉验证。
- 重点公司不能只列名称，必须有分析。
- 结论必须同时包含机会与风险。
- 最终交付必须是钉钉云文档。

## Checklist

- 是否明确行业、地区、时间范围
- 是否关键数据双来源验证
- 是否标注来源和年份
- 是否完成至少一个分析框架
- 是否已创建并更新钉钉文档
