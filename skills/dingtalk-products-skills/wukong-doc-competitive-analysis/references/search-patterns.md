# 竞品分析搜索策略

> 本文件与 `output-template.md` 的报告结构（章节 1-6）和 `SKILL.md` 的 Phase 2-3 保持对齐。

---

## 搜索执行原则

- 对目标产品和每个竞品**并行**执行搜索，不要串行等待
- 定价信息**必须** WebFetch 官网定价页直采，禁止使用博客、评测或第三方摘要
- 每个数据点记录来源 URL 和获取日期
- 关键结论（尤其是市场数据）至少两个来源交叉验证
- 无法获取的数据标注「需补充」，禁止估算或编造

## Phase 2 搜索命令模板

### 1. 竞品识别与定位（对应报告第 1 章）

```
# 竞品全景搜索
web_search: "{目标产品} 竞品 竞争对手 {当前年份}"
web_search: "{目标产品} alternatives competitors {当前年份}"

# 市场定位
web_search: "{产品名} 定位 目标用户 适合场景"
web_search: "{产品名} vs {产品名} 区别 对比 谁更适合"
```

## Phase 3 分析框架

> 与 `SKILL.md` Phase 3 的 A-E 模块对应，与 `output-template.md` 章节 1-6 对应。

### A. 竞争格局（→ 报告第 1 章）
- 竞品分类：直接竞品（核心功能高度重叠）/ 间接竞品（部分场景重叠）/ 潜在威胁（有扩张趋势）
- 威胁等级评估：高 / 中 / 低
- 市场定位象限：选择两个最有区分度的维度（如：产品复杂度 vs 目标客户规模）

### B. 竞品深度画像（→ 报告第 2 章）
- 每个竞品独立分析：成立时间 / 融资阶段 / 核心定位 / 目标用户 / 价值主张 / 最近重大动态
- 提炼 1-2 个独特优势和 1-2 个主要短板

### C. 横向功能矩阵（→ 报告第 3.1 章）
- 行 = 功能模块（按重要性排序，优先列核心功能）
- 列 = 各产品
- 状态统一四档：✅ 支持 / 🔶 部分支持 / ❌ 不支持 / 🔌 第三方集成
- 「🔶 部分支持」必须在表格下方补充具体限制说明

### D. 技术能力对比（→ 报告第 3.3 章，可选）
- 评估维度：部署方式 / 开放 API / 第三方集成数 / 安全认证 / 数据导出格式 / 离线能力
- 适用条件：目标受众为技术团队，或产品有明显技术壁垒时填写

### E. 战略机会识别（→ 报告第 6 章）
- 差距分析：目标产品在哪些维度落后于竞品
- 机会点：按 P0/P1/P2 优先级排列，每条含具体策略 + 可衡量目标 + 时间窗口
- 行动计划：近期（本月内）/ 短期（1-3 个月）/ 中期（3-6 个月），含负责人和完成标志
