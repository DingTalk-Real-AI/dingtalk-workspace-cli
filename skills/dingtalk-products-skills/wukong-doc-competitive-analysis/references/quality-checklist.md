# 竞品分析质量检查清单

> 本清单与 `output-template.md` 的报告结构（章节 0-6）保持对齐，按执行阶段分组检查。

---

## Phase 2：信息采集完成检查

进入 Phase 3 分析前，确认以下项目全部完成：

- [ ] **竞品分类完成**：直接竞品 / 间接竞品 / 潜在威胁已明确区分，威胁等级已评估
- [ ] **产品定位信息已采集**：每个竞品的核心定位、目标用户、价值主张已收集
- [ ] **功能信息已采集**：官网功能页 + 至少一个第三方评测交叉验证
- [ ] **定价已官网直采**：WebFetch 官网定价页，标注获取日期（禁止使用博客/摘要）
- [ ] **市场数据已收集**：用户规模、付费客户数、估值/增速，至少一个可信来源
- [ ] **用户口碑已收集**：评分平台评分 + 典型好差评关键词（G2/知乎/App Store 等）
- [ ] **技术信息已采集**（如适用）：部署方式、API、安全认证、集成数量
- [ ] **缺失数据已标注**：无法获取的数据统一标注「需补充」，未做估算或编造

---

## Phase 3：结构化分析完成检查

进入 Phase 4 写文档前，确认以下分析模块全部完成：

- [ ] **A. 竞争格局**：竞品分类表 + 市场定位象限描述已完成
- [ ] **B. 竞品深度画像**：每个竞品独立画像（定位/优势/短板/最新动态）已完成
- [ ] **C. 横向功能矩阵**：所有产品在相同功能维度下完整评估，使用统一四档（✅/🔶/❌/🔌）
- [ ] **D. 体验评分**：各维度评分有来源依据，「🔶 部分支持」有具体限制说明
- [ ] **E. 技术对比**（如适用）：部署/API/安全/集成/离线能力已横向对比
- [ ] **F. 商业模式**：定价口径统一（每人每月），增长飞轮已分析
- [ ] **G. 市场与口碑**：用户规模和评分数据已填入，来源已标注
- [ ] **H. SWOT 与战略建议**：机会点有优先级（P0/P1/P2），每条行动含具体策略 + 可衡量目标 + 时间窗口

## 成功 / 失败判定

| 判定 | 条件 |
|------|------|
| ✅ 成功 | 报告覆盖竞品分类 + 功能矩阵 + 商业模式 + 市场口碑 + 战略建议五大模块，所有产品在相同维度下评估，每条建议含具体行动和时间窗口，钉钉文档链接已返回 |
| ❌ 失败 | 存在编造数据 / 定价非官网直采 / 建议无具体行动或时间节点 / 文档创建失败且未降级处理 |

## 工具异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| WebFetch 超时或返回 403/404 | 切换为 `web_search` 获取摘要，标注「直采失败，数据来自搜索摘要，建议人工核实」 |
| 搜索结果为空 | 尝试英文名 / 全称 / 缩写扩展搜索；仍无结果则标注「公开信息不足」 |
| 官网无定价页 | 标注「定价需联系销售」，不编造数据 |
| `dws doc create` 失败 | 重试一次；仍失败则将完整报告内容输出到对话中，提示用户手动创建文档 |
| `dws doc update` 失败 | 检查 nodeId 是否正确，重试 `--mode overwrite`；仍失败则分段输出报告内容 |
