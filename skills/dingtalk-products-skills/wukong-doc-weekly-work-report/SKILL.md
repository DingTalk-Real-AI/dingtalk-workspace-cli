# wukong-doc-weekly-work-report

用于生成基于钉钉生态数据的工作周报。主文件只保留触发、时间窗、核心数据源和最短交付流程；详细采集方法和模板见 `references/`。

## 严格禁止 (NEVER DO)

- 不要把聊天回顾当成正式周报交付。
- 不要在没有时间窗口时混入本周外数据。
- 不要把原始日程和任务流水账直接搬进周报。
- 不要编造完成项、行动项或会议结论。
- 不要只写完成事项，不写风险、阻塞和下周计划。
- 不要跳过来源说明，关键结论要能回溯到日程、任务、纪要或文档。

## Tool 总览

| 工具 | 用途 | 安全等级 | 必填参数 | 示例 |
|---|---|---|---|---|
| `dws calendar event list` | 拉取周内日程 | `L1` | `start`, `end` | 查询本周会议 |
| `dws todo task list/get` | 拉取任务和状态 | `L1` | `page`, `size` | 查询完成与进行中任务 |
| `dws minutes` | 获取会议摘要和行动项 | `L1` | `id` | 查纪要摘要 |
| `dws doc search/read` | 获取本周文档活动 | `L1` | `keyword` 或 `node` | 查本周相关文档 |
| `knowledge_answer` | 兜底补背景 | `L1` | `query_list` | 查项目背景 |
| `dws doc create/update` | 生成周报文档 | `L2` | `name` 或 `node` | 创建周报 |

## 意图判断决策树

用户提到“本周周报/一周总结/本周工作汇总”：
- 默认按周一到周日统计

用户给出日期范围：
- 按明确窗口统计，不再默认自然周

用户提到“要汇总会议、任务、纪要、文档”：
- 进入完整周报流程

用户只要“简单周报”：
- 仍至少保留完成项、风险和下周计划三部分

用户提到“整理一下我这周做了什么”：
- 默认按个人工作周报处理

## 易混淆场景

| 如果用户要… | 应使用 |
|---|---|
| 管理层季度或年度经营汇报 | `wukong-doc-business-review-report` |
| 某主题一周资讯动态汇总 | `wukong-doc-topic-trend-report` |
| 行业、赛道或竞争格局研究 | `wukong-doc-industry-research` |

## 核心工作流

1. 确认时间窗口。
2. 拉取四类数据：
   - 日程
   - 任务
   - 纪要
   - 文档活动
3. 提炼而非照搬：
   - 本周完成
   - 关键推进
   - 风险阻塞
   - 下周计划
4. 交付钉钉文档：

```text
-> dws calendar event list --start "2026-03-16T00:00:00+08:00" --end "2026-03-22T23:59:59+08:00"
<- 返回本周日程

-> dws todo task list --page 1 --size 20 --status true
<- 返回本周已完成任务

-> dws minutes get summary --id TASK_UUID
<- 返回会议摘要和行动项

-> dws doc create --name "2026W12 工作周报"
<- 返回文档节点
```

## 上下文传递规则

| 操作 | 从返回中提取 | 用于 |
|---|---|---|
| 日程拉取 | `主题`, `时间`, `参会人` | 关键推进与协作概览 |
| 任务拉取 | `标题`, `状态`, `截止时间` | 完成项与进行中事项 |
| 纪要提取 | `决策`, `行动项`, `负责人` | 风险与后续计划 |
| 文档活动 | `文档名`, `更新时间`, `主题` | 本周产出补充 |
| 文档创建 | `node` 或 `url` | 最终周报更新 |

## MCP 调用约束

- 先拉取原始数据，再做归纳，不要在查询阶段混入总结结论。
- 参数尽量不超过 5 个，嵌套不超过 1 层。
- 先确认时间窗口，再读多源数据，再写周报文档。

## 错误处理

- 缺少纪要时，不要猜会议结论，只能写“已开会，结论待补”。
- 数据过多时，优先保留高影响事项，不做逐条流水账。
- 用户未指定日期时，明确当前采用的自然周范围。
- 某类数据源不可用时，要标注缺失来源，不要静默省略。

## 质量标准

- 周报必须覆盖可用的核心数据源，而不是只写主观总结。
- 输出必须避免流水账，优先提炼完成、风险和计划。
- 关键结论必须能回溯到原始数据源。
- 最终交付必须是钉钉云文档。

## Checklist

- 是否时间窗正确
- 是否覆盖可用的日程、任务、纪要、文档数据
- 是否包含完成项、风险和下周计划
- 是否避免流水账
- 是否已创建并更新钉钉文档
