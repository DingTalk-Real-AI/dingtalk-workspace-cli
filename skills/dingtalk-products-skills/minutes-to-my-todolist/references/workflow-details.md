# 工作流详细说明

## 阶段 1: 获取当前用户
1. 调用 `dws contact user current --format json` 获取当前用户信息
2. 提取 `userId` 和 `name`，用于后续待办匹配
3. 校验: 返回包含有效 userId
4. 失败: 认证失败 → 提示用户重新执行上一条命令（最多重试两次）

## 阶段 2: 获取当天听记列表
依赖: 阶段 1 的用户信息
1. 调用 `dws minutes list mine --max 20 --format json` 获取我的听记
2. 调用 `dws minutes list shared --max 20 --format json` 获取共享给我的听记
3. 合并两个列表，按创建时间筛选出当天的听记
4. 校验: 至少有 1 条当天的听记
5. 失败: 无当天听记 → 告知用户"今天暂无听记"，询问是否查看其他日期

### 决策点: 时间范围选择
- IF 用户指定了日期范围 → 按指定范围筛选
- IF 用户说"最近"/"这几天" → 扩展到最近 3 天
- ELSE → 默认只处理当天

## 阶段 3: 提取听记待办
依赖: 阶段 2 的听记列表
1. 对每条听记调用 `dws minutes get todos --id <taskUuid> --format json`
2. 收集所有待办事项，记录来源听记标题
3. 校验: 至少有 1 条待办
4. 失败: 所有听记均无待办 → 告知用户"听记中未发现待办事项"

## 阶段 4: 筛选我的待办
依赖: 阶段 1 的 userId + 阶段 3 的待办列表
1. 遍历待办列表，检查参与人是否包含当前用户
2. 匹配逻辑: 参与人的 userId 或姓名与当前用户匹配
3. 对无明确负责人的待办，询问用户是否认领
4. 校验: 筛选后至少有 1 条待办
5. 失败: 无我负责的待办 → 告知用户"未发现需要您负责的待办"

### 决策点: 待办认领
- IF 待办无明确负责人且用户选择认领 → 加入待创建列表
- IF 待办无明确负责人且用户选择忽略 → 跳过该待办
- IF 待办有其他负责人但用户要求全部转换 → 加入待创建列表

## 阶段 5: 创建钉钉待办
依赖: 阶段 4 的筛选结果
1. 展示待创建的待办列表，请用户确认
2. MUST: 获得用户确认后再执行创建
3. 对每条待办调用:
   ```bash
   dws todo task create --title "<待办内容>" --executors <userId> --priority 20 --format json
   ```
4. 如有截止时间，添加 `--due` 参数（ISO-8601 格式）
5. 记录每条待办的创建结果（成功/失败）
6. 校验: 至少 1 条创建成功
7. 失败: 全部创建失败 → 报告错误信息

## 阶段 6: 交付结果
依赖: 阶段 5 的创建结果
1. 汇总创建结果：成功 X 条，失败 Y 条
2. 列出已创建的待办标题
3. 提示用户可在钉钉待办中查看和管理
4. 如有失败项，说明失败原因

## 上下文传递表

| 阶段 | 从返回中提取 | 用于 |
|------|-------------|------|
| 阶段 1: `contact user current` | `userId`, `name` | 阶段 4 匹配负责人 |
| 阶段 2: `minutes list mine/shared` | `taskUuid`, `createTime`, `title` | 阶段 3 获取待办 |
| 阶段 3: `minutes get todos` | 待办内容、参与人、时间 | 阶段 4 筛选 + 阶段 5 创建 |
| 阶段 5: `todo task create` | `todoTaskId` | 交付结果展示 |

## 错误处理详表

| 阶段 | 错误 | 处理 | 是否终止流程 |
|------|------|------|-------------|
| 阶段 1 | 认证失败 | 提示用户重新执行上一条命令（最多重试两次） | 是 |
| 阶段 2 | 无当天听记 | 询问是否扩展时间范围 | 否，可继续 |
| 阶段 3 | 某听记获取待办失败 | 跳过该听记，继续处理其他 | 否 |
| 阶段 3 | 所有听记均无待办 | 告知用户，终止流程 | 是 |
| 阶段 4 | 无我负责的待办 | 询问是否转换全部待办 | 否，可继续 |
| 阶段 5 | 单条待办创建失败 | 记录失败原因，继续创建其他 | 否 |
| 阶段 5 | 全部创建失败 | 报告错误信息 | 是 |
| 阶段 5 | userId 无效 | 尝试通过姓名查询 userId | 否，回退查询 |
