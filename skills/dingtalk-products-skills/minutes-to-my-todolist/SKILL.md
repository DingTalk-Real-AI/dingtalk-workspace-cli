# 听记待办转钉钉待办

从当天的钉钉听记中提取待办事项，识别用户需要负责推进的任务，自动创建为钉钉待办，并支持设置每日定时提醒。

## 涉及产品

| 产品 | 用途 | 对应命令 |
|------|------|---------|
| 听记 (minutes) | 获取听记列表和待办事项 | `dws minutes list mine / get todos` |
| 待办 (todo) | 创建待办任务 | `dws todo task create` |
| 通讯录 (contact) | 获取当前用户信息用于匹配负责人 | `dws contact user current` |

## 能力清单

| 动作 | 命令 | 必填参数 | 安全等级 | 说明 |
|------|------|----------|----------|------|
| 获取我的听记列表 | `dws minutes list mine` | -- | 只读 | 获取用户创建的听记，可选 --max 限制数量 |
| 获取共享听记列表 | `dws minutes list shared` | -- | 只读 | 获取他人共享给我的听记 |
| 获取听记待办 | `dws minutes get todos` | --id | 只读 | 获取指定听记中的待办事项 |
| 获取听记信息 | `dws minutes get info` | --id | 只读 | 获取听记基础信息（标题、时间等） |
| 创建待办 | `dws todo task create` | --title, --executors | 写入-需确认 | 创建钉钉待办任务 |
| 获取当前用户 | `dws contact user current` | -- | 只读 | 获取当前登录用户的 userId |

## 操作指引

### 听记获取
1. MUST: 所有 `dws` 命令必须加 `--format json` 以获取可解析输出
2. MUST: 默认获取当天的听记，通过创建时间筛选
3. SHOULD: 同时获取"我创建的"和"共享给我的"听记，确保覆盖完整
4. NOTE: 听记列表返回 `taskUuid`，后续操作均以此为入参

### 待办筛选
1. MUST: 获取当前用户 userId，用于匹配待办中的负责人
2. MUST: 只转换用户需要负责的待办（待办参与人包含当前用户）
3. SHOULD: 如果待办未明确指定负责人，询问用户是否需要认领
4. NOTE: 听记待办返回的参与人信息可能包含姓名，需匹配到 userId

### 待办创建
1. MUST: 待办标题使用听记原文中的待办内容，不做删减
2. MUST: --executors 参数必须使用 userId，不能使用姓名
3. MUST: 创建待办前必须获得用户明确确认
4. SHOULD: 优先级默认设为 20（普通），用户可指定更高优先级
5. SHOULD: 截止时间可从待办原文中提取，无明确时间则不设置
6. NOTE: 创建成功后返回 todoTaskId，可用于后续操作

### 提醒设置
1. NOTE: 钉钉待办本身会在截止时间前提醒，无需额外设置
2. SHOULD: 建议用户设置合理的截止时间以获得系统提醒

## 踩坑提醒

| 错误做法 | 正确做法 | 原因 |
|----------|----------|------|
| 用姓名作为 --executors 参数 | 先查询 userId，再用 userId 作为参数 | API 只接受 userId，姓名会导致创建失败 |
| 直接创建待办不询问用户 | 展示待办列表，获得用户确认后再创建 | 待办是写入操作，必须用户确认 |
| 只获取"我创建的"听记 | 同时获取"我创建的"和"共享给我的"听记 | 用户可能参与他人创建的会议，待办在共享听记中 |
| 遇到空待办列表直接报错 | 告知用户"未发现待办"，询问是否扩展时间范围 | 可能是时间范围太窄，提供友好的替代方案 |
| 重复创建相同待办 | 创建前按标题去重 | 同一待办可能出现在多个听记中（系列会议） |

## 工作流

```
阶段 1: 获取当前用户 → 阶段 2: 获取当天听记 → 阶段 3: 提取待办
                                    ↓
阶段 6: 交付结果 ← 阶段 5: 创建待办 ← 阶段 4: 筛选我的待办
```

| 阶段 | 动作 | 关键产出 | 失败处理 |
|------|------|---------|---------|
| 1 | 获取当前用户 userId | userId, name | 认证失败 → 提示登录 |
| 2 | 获取听记列表（我的 + 共享的） | taskUuid 列表 | 无听记 → 询问扩展范围 |
| 3 | 提取每条听记的待办 | 待办列表 | 无待办 → 终止并告知 |
| 4 | 匹配用户负责的待办 | 筛选后的待办列表 | 无匹配 → 询问是否全部转换 |
| 5 | 用户确认后创建待办 | todoTaskId | 创建失败 → 报告错误 |
| 6 | 汇总结果交付 | 成功/失败统计 | -- |

详细工作流步骤见 [references/workflow-details.md](references/workflow-details.md)

## 使用示例

### 示例 1: 默认场景 — 转换今天的听记待办
**用户说**: "帮我把今天听记里的待办加到钉钉待办"
**执行步骤**:
1. 获取当前用户 userId
2. 获取今天的听记列表（我的 + 共享的）
3. 提取所有听记中的待办事项
4. 筛选出用户需要负责的待办
5. 展示待办列表，请用户确认
6. 用户确认后，批量创建钉钉待办
**期望输出**: "已从 3 条听记中提取 5 条待办，其中 3 条需要您负责。确认转换为钉钉待办吗？\n1. 完成方案初稿（来自：产品评审会）\n2. 协调资源排期（来自：产品评审会）\n3. 输出测试报告（来自：技术对齐会）"

### 示例 2: 查看但不转换
**用户说**: "看看今天会议有什么待办"
**执行步骤**:
1. 同示例 1 的步骤 1-4
2. 展示待办列表，不主动创建
**期望输出**: "今天的听记中共有 5 条待办，其中 3 条需要您负责：\n1. 完成方案初稿\n2. ...\n\n需要我帮您转成钉钉待办吗？"

### 示例 3: 指定时间范围
**用户说**: "把这周听记的待办都整理一下"
**执行步骤**:
1. 识别时间范围为"本周"
2. 获取本周的听记列表
3. 同示例 1 的步骤 3-6
**期望输出**: "本周共有 12 条听记，提取到 8 条您负责的待办。确认转换吗？"

## 错误处理

| 错误类型 | 处理方式 |
|---------|---------|
| 认证失败 | 提示用户重新执行上一条命令（最多重试两次） |
| 无听记/无待办 | 告知用户，询问是否扩展范围 |
| 创建待办失败 | 跳过失败项，继续处理其他，最终汇报失败原因 |

详细错误处理见 [references/workflow-details.md](references/workflow-details.md)

## 注意事项

1. **去重处理**: 同一待办可能出现在多个听记中（如系列会议），创建前需去重
2. **批量限制**: 单次最多创建 20 条待办，超出时分批处理
3. **用户确认**: 创建待办前必须获得用户明确确认，不自动执行
4. **数据隐私**: 只处理用户有权限访问的听记，不访问无权限内容
