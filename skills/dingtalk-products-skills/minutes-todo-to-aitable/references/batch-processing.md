# 批量处理多条听记的策略

## 场景说明

当用户要求处理多条听记的待办时（如"最近一周的会议待办"），需要批量获取、合并去重、统一写入。

## 处理流程

### 1. 获取听记列表

```bash
# 获取最近 N 条听记
dws minutes list mine --max <N> --format json
```

返回中提取每条听记的 `taskUuid` 和 `title`。

### 2. 逐条提取待办

```bash
# 对每个 taskUuid 调用
dws minutes get todos --id <taskUuid> --format json
```

### 3. 合并去重策略

| 判断维度 | 去重规则 |
|----------|----------|
| 内容完全相同 | 保留最近一条 |
| 内容相似（>80%） | 合并并标注来源 |
| 负责人相同 + 内容相近 | 合并为一条 |

## 数据结构

合并后的待办数据结构：

```json
{
  "content": "待办内容",
  "owner": "负责人",
  "sources": [
    {
      "title": "听记标题1",
      "time": "2026-03-20",
      "url": "https://shanji.dingtalk.com/app/transcribes/xxx"
    }
  ],
  "status": "待处理"
}
```

## 写入策略

### 单次写入限制

AI表格 API 单次最多写入 100 条记录。

### 分批写入

```python
# 伪代码
batch_size = 100
for i in range(0, len(todos), batch_size):
    batch = todos[i:i+batch_size]
    # 调用 record create 写入
```

### 错误处理

| 错误 | 处理 |
|------|------|
| 单条写入失败 | 记录失败项，继续其他 |
| 整批写入失败 | 重试 1 次，逐条写入 |
| 超过 100 条 | 自动分批 |

## 用户展示格式

批量处理时的展示格式：

```
找到以下 3 条听记：

| 听记 | 时间 | 待办数 |
|------|------|--------|
| 产品周会 | 3月20日 | 3 条 |
| 技术评审 | 3月19日 | 5 条 |
| 客户复盘 | 3月18日 | 2 条 |

共 10 条待办（去重后 8 条），确认生成汇总表格吗？
```

## 表格命名规则

| 场景 | 命名格式 |
|------|----------|
| 单条听记 | `{听记标题}-待办追踪` |
| 多条听记（同一天） | `{日期}-会议待办汇总` |
| 多条听记（跨天） | `{起始日期}~{结束日期}-会议待办汇总` |
