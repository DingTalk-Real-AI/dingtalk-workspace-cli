# 文档结构分析与内容匹配规则

## 核心原则

**优先融入，其次新增**：分析原文档结构后，将新内容优先融入到匹配的现有章节；只有无合适位置时，才新增段落。

## 结构分析流程

### 1. 获取文档元信息
```bash
dws doc info --node <nodeId> --format json
```
提取：文档标题、类型、创建时间、更新时间

### 2. 读取文档内容
```bash
dws doc read --node <nodeId> --format json
```
获取完整的 Markdown 内容

### 3. 获取块结构
```bash
dws doc block list --node <nodeId> --format json
```
获取文档的块级结构，包含每个块的 blockId、blockType、内容

## 内容匹配规则

### 匹配优先级

1. **章节标题匹配**：听记内容主题与章节标题直接相关
2. **关键词匹配**：听记内容中的关键词出现在某章节
3. **语义匹配**：内容语义与章节主题相近
4. **无匹配**：以上都不满足 → 新增段落

### 常见匹配模式

| 听记内容类型 | 优先融入章节 | 示例 |
|-------------|-------------|------|
| 技术讨论结论 | 技术设计 / 技术方案 / 架构设计 | 缓存策略 → 「技术方案」 |
| 风险和问题 | 风险评估 / 问题清单 / 待解决 | 性能风险 → 「风险评估」 |
| 决策事项 | 决策记录 / 会议决策 / 结论 | 方案选择 → 「决策记录」 |
| 行动计划 | 下一步 / 行动项 / TODO | 跟进事项 → 「下一步」 |
| 需求变更 | 需求描述 / 功能设计 / 需求变更 | 新增功能 → 「需求描述」 |
| 进度更新 | 进度 / 里程碑 / 时间计划 | 完成情况 → 「进度」 |

## 结构识别规则

### 标题层级识别
- `# 标题` → 一级标题（文档主标题）
- `## 标题` → 二级标题（主要章节）
- `### 标题` → 三级标题（子章节）

### 常见文档结构模式

#### 方案文档
```
# 方案标题
## 背景
## 目标
## 方案设计
## 实施计划
## 风险评估
```
**推荐插入位置**：在「实施计划」或「风险评估」之后，或文档末尾

#### 需求文档
```
# 需求标题
## 背景
## 需求描述
## 功能设计
## 验收标准
```
**推荐插入位置**：在「验收标准」之后，或文档末尾

#### 会议纪要
```
# 会议主题
## 参会人
## 议题
## 讨论内容
## 结论与待办
```
**推荐插入位置**：在「结论与待办」之后

#### 通用文档
**推荐插入位置**：文档末尾

## 更新方式决策

### 决策流程

```
听记内容
    ↓
分析每条内容的主题
    ↓
与文档章节逐一匹配
    ↓
┌─────────────┬─────────────┐
│ 找到匹配章节 │ 无匹配章节   │
└─────────────┴─────────────┘
      ↓              ↓
  融入该章节     新增段落
```

### 融入现有章节

当内容与某章节主题匹配时：
1. 找到该章节的最后一个块
2. 在该块之后插入新内容
3. 保持与原章节相同的格式风格

**插入命令**：
```bash
dws doc block insert --node <nodeId> --text "{内容}" --ref-block <blockId> --where after --format json
```

### 新增段落

当无匹配章节时：
1. 在文档末尾新增章节
2. 使用与原文档一致的标题层级
3. 标注来源信息

**插入命令**：
```bash
dws doc block insert --node <nodeId> --heading "{章节标题}" --level 2 --format json
dws doc block insert --node <nodeId> --text "{内容}" --format json
```

### 格式保持规则

| 原文档格式 | 新内容格式 |
|-----------|-----------|
| 有序列表 | 使用有序列表继续编号 |
| 无序列表 | 使用无序列表 |
| 表格 | 在表格中新增行（如适用） |
| 段落 | 使用段落格式 |

## 插入操作规范

### 块插入顺序
1. 先插入新章节标题（heading）
2. 再依次插入内容段落（paragraph/text）
3. 最后插入来源标注

### 块插入命令

**插入标题**：
```bash
dws doc block insert --node <nodeId> --heading "会议补充：{标题}" --level 2 --format json
```

**插入段落**：
```bash
dws doc block insert --node <nodeId> --text "{段落内容}" --format json
```

**在指定位置插入**（在某个块之后）：
```bash
dws doc block insert --node <nodeId> --heading "..." --level 2 --ref-block <blockId> --where after --format json
```

### 格式兼容性

| 内容类型 | 插入方式 |
|---------|---------|
| 标题 | `--heading "..." --level N` |
| 普通段落 | `--text "..."` |
| 列表 | `--text "- 项目1\n- 项目2"` |
| 表格 | `--element '{...}'`（复杂JSON） |
| 引用块 | `--text "> 引用内容"` |

## 安全规则

### 禁止操作
- ❌ 使用 `doc update --mode overwrite`（会清空原文）
- ❌ 使用 `doc block delete`（会删除现有内容）
- ❌ 使用 `doc block update` 修改原有块

### 允许操作
- ✅ 使用 `doc block insert` 追加新块
- ✅ 在文档末尾追加
- ✅ 在指定块之后追加

### 验证检查
插入完成后，应验证：
1. 文档原有内容完整
2. 新增内容正确显示
3. 格式无错乱

## 错误处理

| 场景 | 处理方式 |
|------|---------|
| 找不到指定章节 | 提示用户，建议插入文档末尾 |
| 块插入失败 | 重试 1 次，仍失败则输出内容让用户手动粘贴 |
| 文档只读 | 提示用户检查文档权限 |
| 文档结构异常 | 使用默认策略（文档末尾） |
