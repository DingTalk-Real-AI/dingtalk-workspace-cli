## Writer Agent 执行流程

| 步骤 | 工具 | 动作 |
|------|------|------|
| 1 | `dws doc read` | 读取草稿文档（`--node` 使用完整 `docUrl`），获取全部调研素材 |
| 2 | Agent 自行生成 | 基于素材与模板，**按章节**生成商业计划书 Markdown 内容 |
| 3 | `dws doc create` | 创建空文档：`dws doc create --name "<标题>" --format json`，获取返回的 `docUrl` |
| 4 | `dws doc update` x N | **分章节逐块写入**：每次调用 `dws doc update --node <docUrl> --markdown "<单章内容>" --mode append --format json`，每块控制在 1500 字符以内 |
| 5 | 返回结果 | 返回云文档链接，结束任务 |

> ⚠️ **CRITICAL**：不要尝试通过 `dws doc create --markdown` 一次性传入完整 BP 内容，长内容会导致 `invalidRequest.inputArgs.invalid` 错误。必须先 create 空文档，再分块 update append。

**分块写入示例**：
```bash
# 步骤 3：创建空文档
dws doc create --name "XX 商业计划书" --format json
# 返回 docUrl: https://alidocs.dingtalk.com/i/nodes/xxxxx

# 步骤 4：分章节追加（每块 ≤ 1500 字符）
dws doc update --node "https://alidocs.dingtalk.com/i/nodes/xxxxx" --markdown "# XX 商业计划书\n\n> 融资阶段：A 轮 | 融资金额：5,000 万元\n\n---\n\n## 执行摘要\n\n..." --mode append --format json

dws doc update --node "https://alidocs.dingtalk.com/i/nodes/xxxxx" --markdown "## 一、市场洞察与机会\n\n..." --mode append --format json

# ... 逐章追加，直到全部写完
```
