# 数据质量红线（MUST）

> 以下红线为一票否决项，任一不通过则 BP 不可交付，必须修复后再生成最终文档。

---

## 数据一致性红线

| 红线 | 说明 | 典型违规 |
|------|------|---------|
| SAM/SOM 必须基于自身定价计算 | SAM = 目标客户数 × **自身产品客单价**，不可用行业整体 IT 支出替代 | 产品定价 3-15 万/年，却用"年均软件支出 50-100 万"算 SAM |
| SOM 必须与 ARR 预测可达 | SOM 目标与 3 年 ARR 预测差距不超过 10 倍，否则需解释路径 | SOM 写 7.5 亿，ARR 预测仅 5000 万 |
| 里程碑与财务预测时间线对齐 | 融资里程碑中的客户数/ARR 必须与年度财务预测表一致 | 里程碑写"18 个月 60 家"，年度表 2026 年底才 25 家 |
| 效果数据必须标注验证条件 | 产品效果数据（如"降损 30%"）必须注明样本量、测试周期、客户类型 | 仅写"实测误差率<5%"，无样本量和测试条件 |
| 退出估值必须有倍数依据 | 投资回报测算中的估值必须标注 PS/PE 倍数及可比公司 | 直接写"估值 8 亿"，无倍数推导 |

## 数据来源红线

| 红线 | 说明 |
|------|------|
| 市场规模至少 2 个独立来源交叉验证 | 仅引用 1 个报告不可接受；差异超 30% 需注明口径差异 |
| 竞品数据必须标注来源 | 竞品营收、客户数、融资等信息需注明出处（官网/报道/招股书） |
| 行业痛点数据必须标注来源 | "损耗率 8-12%"等关键数据需注明调研机构或报告名称 |
| 投资回报情景概率需有依据 | 不可凭空分配概率（如 60%/25%/15%），需说明参考基准 |

## 团队章节红线

| 红线 | 说明 |
|------|------|
| 禁止全占位符团队章节 | 核心团队（CEO/CTO/COO）至少提供：前公司+职级+行业年限；若用户未提供，主 Agent 必须主动询问 |
| 团队优势必须有实证 | "平均 10 年经验"等断言必须有具体人员背景支撑，不可空洞概括 |
| 查不到团队信息时的处理 | 标注"需补充：创始人背景（前公司、职级、行业经验）"，并在执行摘要中注明"团队信息待完善" |
