# 商业计划书质量检查清单

---

## 主 Agent — 市场数据检查项

- [ ] TAM/SAM/SOM 三级市场规模已量化（至少 2 个来源）
- [ ] 行业增长率（CAGR）已获取并标注来源
- [ ] 目标用户画像已明确（行业、规模、决策者、预算）
- [ ] 市场痛点已验证（有调研数据或客户反馈支撑），**痛点数据已标注来源**
- [ ] 自上而下与自下而上估算已交叉验证
- [ ] **SAM 使用自身产品定价区间计算**（非行业整体 IT 支出）
- [ ] **SOM 与 3 年 ARR 预测差距不超过 10 倍**（否则需解释路径）

## 主 Agent — 竞品数据检查项

- [ ] 主要竞品已识别（至少 3-5 家）
- [ ] 竞品核心信息已收集（产品、定价、融资、客户数），**每项数据已标注来源**
- [ ] 竞品对比矩阵已建立（维度统一、评分有依据）
- [ ] 差异化壁垒已明确（技术、数据、场景、切换成本）

## 主 Agent — 财务数据检查项

- [ ] 行业财务基准已获取（毛利率、LTV/CAC、估值倍数）
- [ ] 可比融资案例已收集（至少 2-3 个）
- [ ] 财务预测已建立（3-5 年收入与盈利）
- [ ] 单位经济模型已计算（CAC、LTV、回收期）
- [ ] 融资需求与资金用途已拆分
- [ ] **融资里程碑与年度财务预测表时间线一致**（客户数/ARR 无矛盾）
- [ ] **投资回报测算中的估值已标注 PS/PE 倍数及可比公司依据**

## 主 Agent — 内部资料检查项

- [ ] 产品信息已获取（功能、技术架构、路线图）
- [ ] 经营数据已获取（客户数、ARR、NDR、NPS）
- [ ] 团队信息已获取（核心成员背景、组织规划）
- [ ] **团队章节非全占位符**：核心团队至少提供前公司+职级+行业年限；若缺失已主动询问用户
- [ ] 已有内部分析/文档已检索
- [ ] **产品效果数据已标注验证条件**（样本量、测试周期、客户类型）

## 主 Agent — 分析检查项

- [ ] 商业模式画布已完成（价值主张、收入模式、成本结构）
- [ ] SWOT 分析已完成（含战略应对）
- [ ] 增长飞轮/GTM 策略已明确
- [ ] 风险矩阵已建立（风险+影响+概率+对策）

## 常见失败模式与修复

| 失败模式 | 修复方法 |
|---------|---------|
| 市场规模论证薄弱 | 构建 TAM/SAM/SOM 三级体系，≥ 2 个独立来源交叉验证，自上而下+自下而上双向估算 |
| SAM/SOM 与产品定价脱节 | SAM = 目标客户数 × 自身客单价；SOM 与 3 年 ARR 差距 ≤ 10 倍 |
| 竞品分析缺乏深度 | 建立加权评分矩阵，每项评分有依据来源；用 fetch 深度阅读竞品官网及评测 |
| 盈利模式描述模糊 | 按商业模式画布填写，收入拆分为订阅/服务/增值，给出定价区间和占比 |
| 财务预测缺乏可信度 | 对标行业 benchmark，增速逐年递减符合 S 曲线，注明关键假设 |
| 里程碑与财务预测不匹配 | 先定年度财务表，再推导里程碑，确保时间线和数值一致 |
| 团队信息过于简单 | 核心成员须含前公司+职级+关键业绩；缺失时主动询问用户 |
| 产品效果数据缺失验证条件 | 标注样本量+测试周期+客户类型（如"12 家客户实测 6 个月"） |
| 投资回报测算缺少依据 | 标注 PS/PE 倍数及可比公司；情景概率需说明参考基准 |
| 风险分析流于形式 | 风险矩阵含影响程度+概率+可执行对策，非口号式表述 |
