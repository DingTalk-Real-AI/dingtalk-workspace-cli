## 4. 每个测试文件用意详表

### 4.1 `test_01_user.py`（18 条）— 人员查询主路径 + 重名消歧结构保护

- **TestUserSearch 系列**：`user search --query/--keyword` 双参数别名、边界值（空 / 单字 / 超长）、分页 `--page/--size`。
- **TestUserGet 系列**：`user get --ids` 批量取人，验证 userId/name/depts 字段齐整。
- **TestUserGetSelf**：`user get-self` 当前登录态身份返回，手机号掩码。
- **TestUserSearchMobile**：`--mobile` 精确匹配。
- **TestDisambiguation** — 重名消歧系统性保护：
  - `search` 结果无 `depts` 字段（强制触发 get 补齐）
  - `search` 结果不出现 `orgEmployeeModel` 嵌套（未来改嵌套就会漏检）
  - 两步法合流：search → 多命中 → get --ids → 根据 depts 区分同名人

### 4.2 `test_02_dept.py`（15 条）— 部门四命令与作用域语义

- **TestDeptSearch**：关键词搜索部门；
- **TestDeptGetInfo**：`get-info --id` 单部门详情；
- **TestDeptListChildren**：`list-children --id` 子部门列表（只返直属）；
- **TestDeptListMembers**：成员查询，3 条锁死：
  - 必填 `--ids` 校验
  - 批量 `--ids "A,B"` 合并返回
  - **不递归子部门**（反证法；数据不具备时 skip）

### 4.3 `test_03_user_profile_dismission.py`（15 条）— HR 花名册 + 离职员工

- **TestContactUserDismission**：`dismission search` 分页取离职名单，`--page/--size/--hide-retirement/--hide-partner` 组合。
